# LifePocket / 生活口袋手机 App 使用手册

## 1. LifePocket 是什么

LifePocket / 生活口袋 是一个 AI 日常生活整理 App。你可以把截图、小票、账单、预约、购物清单、旅行信息或一段文字交给 App，App 会调用你自己配置的书生模型 API，整理成生活卡片、提醒、消费记录和清单。

App 不内置 API Token。首次使用前，你需要在设置页填写自己的书生模型 API Token。

## 2. 初次使用流程

1. 打开 App。
2. 进入“设置”页。
3. 填写 API Endpoint。
4. 填写 Model。
5. 填写 API Token。
6. 点击“测试连接”。
7. 回到“上传”页开始识别。

默认配置可以使用：

- API Endpoint：https://chat.intern-ai.org.cn/api/v1/chat/completions
- Model：intern-latest
- API Token：需要你自己填写，不能使用示例 Token。

## 3. 如何上传识别

在“上传”页，你可以选择图片、拍照，或直接粘贴文字。适合识别的内容包括小票、账单、预约、购物订单、旅行信息、保修信息和普通待办。

识别结果会生成生活卡片。如果一张图片或一段长文本中有多条独立记录，App 会尽量生成多张生活卡片，你可以单独保存某一张，也可以一键保存全部。

## 4. 生活卡片是什么

生活卡片用于保存日常事项和结构化信息，例如消费、账单、预约、旅行、待办、购物订单和保修信息。卡片可以标记状态、添加提醒、保留历史记录，也可以归档。

## 5. 生活卡片类型说明

- expense：已经发生的消费记录，例如小票、支付截图、消费账单。
- bill：待支付账单，例如水电费、信用卡还款、物业费。
- appointment：预约、会议、看病、课程、活动。
- shopping：购物、订单、快递、采购事项。
- travel：机票、酒店、行程、旅行计划。
- warranty：发票、保修卡、商品保修。
- todo：普通待办事项。
- note：普通备忘。
- unknown：无法明确分类的信息。

## 6. 卡片状态说明

- active：进行中 / 待处理。表示这张卡片还需要关注，例如未支付账单、未完成待办、即将到来的预约。
- done：已完成 / 已支付。表示事情已经处理完，例如账单已支付、待办已完成、预约已结束。
- archived：已归档。表示这张卡片暂时不需要出现在主要列表中，但仍保留在本地历史中。

状态对页面展示的影响：

- 首页默认优先显示 active 卡片。
- done 卡片可以作为历史记录保留。
- archived 卡片不应干扰首页，但不应被删除。
- 删除是永久移除，归档不是删除。

## 7. 如何使用清单功能

进入“清单”页，输入生活场景，例如“周末露营”“搬家准备”“去医院看牙”。AI 会生成一份实用清单，并保存到历史清单。

你可以查看历史清单，进入清单详情页修改标题、说明和清单项。清单项支持添加、删除、编辑、填写数量、填写分类，以及勾选 / 取消勾选。

## 8. 隐私与安全

App 不内置 API Token。Token 由你自己填写，并保存在本地安全存储中。上传图片或文本调用模型时，内容会发送给你配置的模型接口。

不建议上传身份证、银行卡、完整合同、敏感票据等高敏感信息。App 界面只显示脱敏后的 Token，不应在日志中打印完整 Token。

## 9. 常见问题 FAQ

### 为什么识别失败？

可能是网络不可用、Endpoint 填写错误、Token 无效、模型暂时不可用，或图片过大。请先在设置页测试连接，再换一张更清晰、更小的图片重试。

### 为什么提示没有 API Token？

你还没有在设置页保存 Token，或者已经清除了本地 Token。请重新填写并保存。

### 为什么图片识别结果不准确？

截图模糊、文字太小、长截图内容太复杂，都会影响识别。可以裁剪重点区域，或在上传页补充文字说明。

### 为什么一张小票只识别出部分内容？

同一次消费的小票通常会合并成一张消费卡片。如果图片中确实有多笔独立消费，但只识别出部分内容，可以裁剪后分次识别，或补充文字说明。

### 如何修改 AI 生成的清单？

进入清单详情页，直接修改标题、说明、事项内容、数量和分类。勾选和编辑会立即保存到本地。

### 归档和删除有什么区别？

归档只是让卡片不再干扰首页主要列表，数据仍保留在本地。删除是永久移除。

### 如何清除 API Token？

进入设置页，点击“清除 Token”。清除后再次识别前需要重新填写 Token。
